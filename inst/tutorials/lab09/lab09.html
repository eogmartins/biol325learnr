Exercise 1.5: Assess the fit of the interaction model

Before you look at the model parameter estimates, you have to ensure that the model is a good fit to the data. In GLMs, the assumptions (correct response distribution, appropriate link function, correctly specified predictors, and independence of observations, etc.) together define how the data are generated, so they are best evaluated through overall model fit rather than checked one by one.

The R package DHARMa does this with functions that simulate data from the fitted model and compare it to what was actually observed to compute standardized quantile residuals. If the model is a good fit to the data, the quantile residuals should be uniformly distributed between 0 and 1 with no patterns. Therefore, when DHARMa diagnostics indicate a good fit, it suggests there is no evidence that the GLM assumptions are violated. Conversely, systematic patterns in the residuals indicate that at least one of those underlying assumptions is not being met.

Exercise 1.8: Compare the models

In this exercise, we compare the main effects and interaction models using the function anova() with a likelihood ratio test (LRT). The LRT evaluates whether the more complex model (i.e. with interaction) provides a significantly better fit to the data. It does this by comparing the deviance of the two models, which is a measure of model fit based on how well the model explains the observed data (lower deviance indicates better fit).

The test statistic is the difference in deviance between the models and follows a chi-squared distribution. The null hypothesis is that the interaction term has no effect (its coefficient(s) are equal to zero). A small P-value would lead us to reject this null hypothesis, indicating that the interaction improves the model and that the effect of one predictor depends on the level of the other. However, a large P-value indicates that there is no evidence that the interaction improves model fit, and that the simpler main effects model is sufficient to describe the data.

Exercise 1.10: Compute model predictions

As you may have noticed, interpreting the Bernoulli GLM directly from its coefficients can be challenging because they are expressed on the log-odds (logit) scale. Instead, a more intuitive approach is to examine the model’s predicted probabilities of nest failure, which allow us to understand how the response changes across predictor values in a meaningful way.

To do this, we first create a new data frame (pred_grid) with a prediction grid that contains a sequence of values for PpnGrass spanning its observed range, combined with all levels of Habitat. This allows us to generate predictions across the full gradient of PpnGrass for each habitat type. Examine the commands below to create this prediction grid.

pred_grid <- expand.grid(
  PpnGrass = seq(min(mallard_clean$PpnGrass),
                 max(mallard_clean$PpnGrass),
                 length.out = 200),
  Habitat  = levels(mallard_clean$Habitat)
)

We then use the predict() function with type = "link" to obtain predictions on the logit scale, along with their standard errors (se.fit = TRUE), like so

pred_link <- predict(mod_main, newdata = pred_grid, type = "link", se.fit = TRUE)

Importantly, the confidence intervals must be calculated on the logit scale, where the model is linear and the standard errors are defined. We construct these intervals by adding and subtracting 1.96 times the standard error from the fitted values.

Finally, we transform the fitted values and the confidence limits to the probability scale using the inverse logit function plogis(). This gives us the predicted probability of nest failure (fit), along with lower (lwr) and upper (upr) 95% confidence intervals that can be more easily interpreted and visualized.

Examine the commands below.

pred_grid <- pred_grid |>
  mutate(
    fit = plogis(pred_link$fit),
    lwr = plogis(pred_link$fit - 1.96 * pred_link$se.fit),
    upr = plogis(pred_link$fit + 1.96 * pred_link$se.fit)
  )

The eagles data frame is available from R package MASS, which includes data and functions associated with the book Modern Applied Statistics with S by W. N. Venables and B. D. Ripley.

You will now fit the Binomial GLM with the base R function glm(). Just like you did with the Bernoulli GLM in the previous section, you have to set family = binomial.

You will fit the same model the authors of the original publication used, which included all three main effects (P, A, and V) as well as an interaction between P and A. In R, this Binomial GLM is fit with the following command:

mod_eagles_inter <- glm(cbind(y, n - y) ~ P * A + V, family = binomial, data = eagles_clean)

Note the response term (i.e., the term before ~). For a Binomial GLM, you have to provide glm() the response as a matrix containing the number of successes and the number of failures, in this order. This is achieved with function cbind() to combine the number of successful food-stealing attemps (y) and the number of failed attempts (n - y, the the total number of attempts minus the number of successful ones).

Also, note that internally R will expand P * A + V to P + A + V + P:A.
